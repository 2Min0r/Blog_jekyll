  1. 프로그래머 첫걸음 01, 02, 03

    • 피즈버즈(FizzBuzz) 테스트란?

      • 프로그래밍 관련 직종에 지원하는 사람 200명 중 199명이 코드를 전혀 작성하지 못해서 만든 테스트

    • 코드 맨 앞에 주석을 이렇게 달자!

      # [파일명.py] 			- 파일명
    # http://url		  - 코드가 있는 주소
    # coding = utf-8

    • 고정폭 폰트

      • 글자의 폭이 일정하므로, 들여쓰기나 코드의 정렬 상태가 잘 나타나서 프로그램 관련 텍스트에 자주 쓰임

    • http://tinyurl.com/

      • 긴 url 주소를 짧게 줄여주는 사이트. 복사해서 붙여넣기만 하면 생성된다. 해당 책의 예시 url이 tinyurl을 사용하여 담겨있다.

    • 헤더(header)와 스위트(suite)

      • 두 개 이상의 행으로 구성되는 절에서 첫 행을 헤더, 그 이하를 스위트라고 부른다.

        x = 100
    if x == 10:
        print("10!")
    else:
        print("I don't know!")

      • x는 100이므로 if 스위트는 실행되지 않으며, else 스위트는 실행된다.

    • 옵션 매개변수

      • 옵션 매개변수가 전달되지 않으면 기본값을 사용함
      def f(x = 2):
        return x**x
    print(f())			# 4
    print(f(4))			# 256
    
      def add_it(x, y = 10):
        return x + y
    print(add_it(2))	# 12

    • 예외처리

      • try문에서 정의한 변수를 except문에서 사용할 경우 변수가 정의되기 전에 예외가 일어날 수 있어 정의하지 않은 변수를 사용하게 될 수 있다.(NameError)

    • 독스트링

      • 함수를 정의할 때 함수 맨 위에 특별한 주석을 남기는 것.

      • 각 매개변수가 어떤 데이터 타입이어야 하는지, 함수가 무슨 일을 하는지 문서화하는 역할

        def add(x, y):
        """
        Returns x + y.
        :param x: int.
        :param y: int.
        :return: int sum of x and y.
        """
        return x + y

  1. 프로그래머의 자세

    • 첫 번째 버전이 동작한다고 해서 멈춰서는 안 되며, 코드가 아름다워 보일 때까지 계속 리팩토링하고 정제해야 한다.
    • 훌륭한 프로그래머는 돈이나 명성을 위해 프로그래밍하는 것이 아니라 프로그래밍 자체가 재미있어서 한다.
    • 함수는 한 가지 일만 해야 하고, 그 일을 잘 해야 한다.
    • 불필요하게 변수를 사용하지 않는다. 나중에 사용하려고 하는 데이터만 변수에 저장해야 한다.
